# Test Driven Development Plan

## Goal

Develop features starting from tests to ensure correctness and clarity.

## Step 1: Patient creation

Test:
- should create patient successfully
- should return validation error for invalid input

## Step 2: Audit log

Test:
- should create audit log entry when patient is created

## Principles

- Write tests first
- Implement minimal code to pass tests
- Refactor if needed

## Why TDD?

- Ensures correctness
- Prevents overengineering
- Makes code easier to reason about
